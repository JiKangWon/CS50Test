1. Khái niệm (Concept)

Định nghĩa

Mảng hai chiều là một mảng mà các phần tử của nó là mảng một chiều. Nó giống như một bảng gồm có dòng và cột, được đánh dấu vị trí bằng các chỉ số bao gồm chỉ số dòng và chỉ số cột.

Mảng hai chiều hay còn được gọi là ma trận.

Một mảng A có m dòng và n cột (ký hiệu $A_{m,n}$ ) sẽ có các chỉ số dòng chạy từ 0 đến m-1 và các chỉ số cột chạy từ 0 đến n-1.

Các khái niệm với Ma Trận Vuông

Ma trận vuông là ma trận có số dòng bằng số cột (ví dụ, n x n). Đối với ma trận vuông, ta có các khái niệm:

  • Đường chéo chính: Gồm các phần tử có chỉ số dòng = cột.
  • Đường chéo phụ: Gồm các phần tử có chỉ số dòng + cột = n - 1.
  • Ma trận tam giác trên: Các phần tử nằm trên đường chéo chính, có dòng < cột.
  • Ma trận tam giác dưới: Các phần tử nằm dưới đường chéo chính, có dòng > cột.
  • Các khái niệm khác: Ma trận đơn vị , Ma trận đối xứng.

2. Khai báo (Declaration)

Khai báo kiểu mảng (Sử dụng typedef)

Ta có thể định nghĩa một kiểu dữ liệu mảng 2 chiều mới bằng typedef.

Cú pháp: 

typedef <kiểu cơ sở> <tên kiểu>[<N1>][<N2>];
  • <N1> và <N2> là số lượng phần tử tối đa của mỗi chiều.

Ví dụ: 

typedef int MaTran[3][4];

Khai báo biến mảng

Có hai cách để khai báo một biến mảng 2 chiều:

1. Tường minh (Explicit)

Khai báo trực tiếp kiểu và kích thước.

Cú pháp: 

<kiểu cơ sở> <tên biến>[<N1>][<N2>];

Ví dụ: 

int a[10][20], b[10][20]; 
int c[5][10];

2. Không tường minh (Implicit - Thông qua kiểu)

Sử dụng một kiểu typedef đã định nghĩa trước đó.

Cú pháp: 

typedef <kiểu cơ sở> <tên kiểu>[<N1>][<N2>]; 
<tên kiểu> <tên biến>; 
<tên kiểu> <tên biến 1>, <tên biến 2>;

Ví dụ: 

typedef int Matran10x20[10][20]; 
typedef int Matran5x10[5][10]; 

Matran10x20 a, b; 
Matran5x10 c;

3. Truy xuất dữ liệu kiểu mảng

Truy xuất đến một phần tử

Ta truy xuất đến một phần tử cụ thể thông qua chỉ số dòng và chỉ số cột.

Cú pháp: 

<tên biến mảng>[<giá trị chỉ số dòng>][<giá trị chỉ số cột>]

Ví dụ: Cho mảng int a[3][4];

  • Truy xuất hợp lệ: a[0][0], a[0][1], ..., a[2][2], a[2][3]. (Chỉ số dòng từ 0-2, chỉ số cột từ 0-3).
  • Truy xuất KHÔNG hợp lệ: a[-1][0], a[2][4], a[3][3]. (Vượt ra ngoài biên mảng đã khai báo).

Gán dữ liệu kiểu mảng

Không được sử dụng phép gán thông thường (=) để sao chép toàn bộ mảng này sang mảng khác.

Phép gán = ; là SAI.

Để sao chép mảng, ta phải gán trực tiếp giữa các phần tử, thường dùng 2 vòng lặp lồng nhau.

Ví dụ: Sao chép mảng a sang mảng b (cùng kích thước 5x10). 

int a[5][10], b[5][10]; 
// b = a; // Lệnh này là SAI!

// Cách làm đúng:
int i, j; 
for (i = 0; i < 5; i++) 
{
    for (j = 0; j < 10; j++) 
    {
        b[i][j] = a[i][j]; 
    }
}

Truyền mảng cho hàm

Khi khai báo hàm, tham số kiểu mảng 2D được viết tương tự như khai báo biến mảng.

Lưu ý quan trọng: Khi truyền mảng 2D, bạn có thể bỏ qua kích thước chiều thứ nhất (số dòng), nhưng BẮT BUỘC phải chỉ định kích thước chiều thứ hai (số cột).

Việc truyền mảng cho hàm thực chất là truyền địa chỉ của phần tử đầu tiên của mảng. Do đó, mọi thay đổi nội dung của mảng bên trong hàm sẽ được giữ lại sau khi hàm kết thúc.

Các cách khai báo tham số mảng 2D trong hàm (ví dụ mảng có 100 cột):

void HamViDu(int a[50][100]); 
// Hoặc
void HamViDu(int a[][100]); 
// Hoặc dùng cú pháp con trỏ
void HamViDu(int (*a)[100]);

Truyền số phần tử thực sự

Vì mảng được khai báo với kích thước tối đa (ví dụ 50x100), ta phải truyền số lượng phần tử (số dòng, số cột) thực sự đang sử dụng qua các biến riêng (ví dụ m và n).

Ví dụ đầy đủ:

#include <iostream>
using namespace std;

// Giả sử mảng có tối đa 100 cột
const int MAXC = 100;
const int MAXD = 50;

// Khi nhập, m và n cần thay đổi nên dùng tham chiếu (&)
void NhapMaTran(int a[][MAXC], int &m, int &n) { 
    cout << "Nhap so dong, so cot: ";
    cin >> m >> n;
    // ... (code nhập mảng)
}

// Khi xuất, m và n chỉ đọc nên không cần tham chiếu
void XuatMaTran(int a[][MAXC], int m, int n) { 
    // ... (code xuất mảng)
}

int main() { 
    int a[MAXD][MAXC]; 
    int m, n; 
    
    NhapMaTran(a, m, n); 
    XuatMaTran(a, m, n); 
    
    return 0;
}

4. Một số bài toán cơ bản

Dưới đây là cài đặt cho một số hàm xử lý mảng 2 chiều cơ bản.

Quy ước chung

Chúng ta sử dụng các hằng số để định nghĩa kích thước tối đa của mảng. 

#define MAXD 50 
#define MAXC 100

Và sử dụng một số hàm phụ trợ:

Hàm HoanVi (Lưu ý: code trong slide dường như có lỗi logic, tham số x nên là tham chiếu)
void HoanVi(int x, int &y) 
{
    int tam = x; 
    x = y; 
    y = tam; 
}
Hàm LaSNT (Kiểm tra số nguyên tố) 
int LaSNT(int n) 
{
    int i, dem = 0; 
    for (i = 1; i <= n; i++) 
    {
        if (n % i == 0) 
            dem++; 
    }
    if (dem == 2) 
        return 1; 
    else
        return 0; 
}

1. Nhập và Xuất Ma Trận

Hàm Nhập Ma Trận

Ý tưởng: Cho trước mảng 2 chiều với kích thước tối đa. Nhập vào số dòng m và số cột n thực sự. Dùng 2 vòng lặp for lồng nhau để nhập từng phần tử từ a[0][0] đến a[m-1][n-1].

Hàm Nhập Ma Trận
void NhapMaTran(int a[][MAXC], int &m, int &n) 
{
    cout << "Nhap so dong, so cot cua ma tran: "; 
    cin >> m >> n; 
    
    int i, j; 
    for (i = 0; i < m; i++) 
    { 
        for (j = 0; j < n; j++) 
        {
            cout << "Nhap a[" << i << "][" << j << "]= "; 
            cin >> a[i][j]; 
        } 
    } 
}

Hàm Xuất Ma Trận

Ý tưởng: Dùng 2 vòng lặp for. Vòng lặp ngoài duyệt các dòng (0 đến m-1). Vòng lặp trong duyệt các cột (0 đến n-1) và in ra giá trị a[i][j]. Xuống dòng mới ("\n") sau khi in xong mỗi hàng.

Hàm Xuất Ma Trận
void XuatMaTran(int a[][MAXC], int m, int n) 
{
    int i, j; 
    for (i = 0; i < m; i++) 
    { 
        for (j = 0; j < n; j++) 
        {
            cout << a[i][j] << " "; 
        }
        cout << "\n"; 
    } 
}

2. Tìm kiếm một phần tử

Yêu cầu: Tìm xem phần tử x có nằm trong ma trận a (kích thước mxn) hay không.

Ý tưởng: Duyệt từng phần tử của ma trận. Nếu phần tử đang xét a[i][j] bằng x thì trả về 1 (tìm thấy). Nếu duyệt hết ma trận mà không tìm thấy, trả về 0.

Hàm Tìm Kiếm 
int TimKiem(int a[][MAXC], int m, int n, int x) 
{
    int i, j; 
    for (i = 0; i < m; i++) 
    {
        for (j = 0; j < n; j++) 
        {
            if (a[i][j] == x) 
                return 1; 
        }
    }
    return 0; 
}

3. Kiểm tra tính chất (Ví dụ: Ma trận toàn số nguyên tố)

Yêu cầu: Kiểm tra ma trận a (kích thước mxn) có phải là ma trận toàn các số nguyên tố hay không.

Có nhiều cách tiếp cận:

  • Cách 1: Đếm số lượng SNT. Nếu số lượng này bằng m * n thì trả về 1.
  • Cách 2: Đếm số lượng số KHÔNG phải SNT. Nếu số lượng này bằng 0 thì trả về 1.
  • Cách 3 (Hiệu quả nhất): Duyệt ma trận, tìm xem có phần tử nào KHÔNG phải SNT hay không. Nếu tìm thấy dù chỉ 1 số, lập tức trả về 0 (không toàn SNT). Nếu duyệt hết mảng mà không tìm thấy, trả về 1.
Hàm Kiểm Tra (Cách 3) 
int KiemTraC3(int a[][MAXC], int m, int n) 
{
    int i, j; 
    for (i = 0; i < m; i++) 
    {
        for (j = 0; j < n; j++) 
        {
            if (LaSNT(a[i][j]) == 0)  // Nếu a[i][j] không phải SNT
                return 0;  // Kết luận ngay
        }
    }
    return 1;  // Đã duyệt hết mà không thấy số nào vi phạm
}

4. Tính tổng các phần tử

Ý tưởng chung: Duyệt ma trận và cộng dồn các phần tử có tọa độ (dòng, cột) thỏa mãn yêu cầu.

Hàm tính tổng trên dòng d 
int TongDong(int a[][MAXC], int m, int n, int d) 
{
    int j, tong; 
    tong = 0; 
    // Duyệt các cột trên dòng d
    for (j = 0; j < n; j++) 
    {
        tong = tong + a[d][j]; 
    }
    return tong; 
}
Hàm tính tổng trên cột c 
int TongCot(int a[][MAXC], int m, int c) 
{
    int i, tong; 
    tong = 0; 
    // Duyệt các dòng trên cột c
    for (i = 0; i < m; i++) 
    {
        tong = tong + a[i][c]; 
    }
    return tong; 
}

Các bài toán sau đây áp dụng cho ma trận vuông (nxn).

Hàm tính tổng đường chéo chính 
int TongDCChinh(int a[][MAXC], int n) 
{
    int i, tong; 
    tong = 0; 
    for (i = 0; i < n; i++) 
    {
        tong = tong + a[i][i];  // Phần tử có dòng = cột
    }
    return tong; 
}
Hàm tính tổng các phần tử trên đường chéo chính 
int TongTrenDCChinh(int a[][MAXC], int n) 
{
    int i, j, tong; 
    tong = 0; 
    for (i = 0; i < n; i++) 
    {
        for (j = 0; j < n; j++) 
        {
            if (i < j)  // Phần tử có dòng < cột
                tong = tong + a[i][j]; 
        }
    }
    return tong; 
}

int TongTrenDCChinh(int a[][MAXC], int n)  // Tên hàm trong slide có vẻ là typo
{
    int i, j, tong;

    for (i = 0; i < n; i++) 
    {
        for (j = 0; j < n; j++)

            if (i > j)  // Phần tử có dòng > cột

Hàm tính tổng đường chéo phụ 
int TongDCPhu(int a[][MAXC], int n) 
{
    int i, tong; 
    tong = 0; 
    for (i = 0; i < n; i++) 
    {
        tong = tong + a[i][n - i - 1];  // Phần tử có dòng + cột = n - 1
    }
    return tong; 
}

5. Tìm giá trị lớn nhất (Max) của Ma Trận

Yêu cầu: Tìm giá trị lớn nhất trong ma trận a (kích thước mxn).

Ý tưởng:

  1. Giả sử giá trị max hiện tại là phần tử đầu tiên a[0][0].
  2. Lần lượt kiểm tra tất cả các phần tử còn lại của ma trận (dùng 2 vòng lặp).
  3. Nếu tìm thấy phần tử a[i][j] nào lớn hơn max, cập nhật max = a[i][j].
  4. Sau khi duyệt hết mảng, max chính là giá trị lớn nhất.
Hàm tìm Max 
int TimMax(int a[][MAXC], int m, int n) 
{
    int i, j, max; 
    max = a[0][0]; 
    
    for (i = 0; i < m; i++) 
    {
        for (j = 0; j < n; j++) 
        {
            if (a[i][j] > max) 
            {
                max = a[i][j]; 
            }
        }
    }
    return max; 
}
